## 📊 Prompt 优化（极简版）

### 优化原则
**交由云端处理的内容**：
- ❌ 研究视角（trader/researcher/fund-manager 等）
- ❌ 问题类型（analytical/comparative/predictive 等）
- ❌ 复杂度等级（simple/medium/complex）
- ❌ 输出结构（执行摘要 → ...）
- ❌ 核心实体（公司/行业/概念）

**保留的内容**：
- ✅ 研究问题
- ✅ 研究目标（整合了研究框架）
- ✅ 信息源标准

## 📍 进度通知逻辑修正

### 问题

之前进度通知显示的是 `progress.stage`（阶段），但实际应该优先显示 `progress.subtask`（子任务名称）。

**服务端流式输出示例**：
```json
{
  "stage": "researcher",
  "subtask": "智能体 researcher 推理中...",
  "percent": 40
}
```

**用户应该看到的是**：
- ✅ 智能体 researcher 推理中...（subtask，具体任务）
- ❌ researcher（stage，抽象阶段）

---

### 修复

**文件**: `src/core/research.js`

**修改内容**:
```javascript
// 修复前
const displayMessage = progress.stage || progress.subtask || '';

// 修复后
// 进度显示：优先使用 subtask（子任务名称），降级使用 stage
const displayMessage = progress.subtask || progress.stage || '';
```

**通知函数已正确处理**（`src/ui/cards.js`）：
```javascript
// 优先显示子任务
const displayStage = (subtask && subtask.trim() !== '') 
  ? subtask 
  : (currentStage && currentStage.trim() !== '')
    ? currentStage
    : '研究进行中...';
```
