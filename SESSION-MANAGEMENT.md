# 会话管理策略

> **版本**: 1.0
> **适用场景**: 任务级多轮对话（2-6 小时）

---

## 🎯 核心原则

1. **文件 > 脑记** - 所有重要信息写入文件
2. **阶段 > 连续** - 长任务分阶段，每阶段独立会话
3. **主动 > 被动** - 上下文超限前主动 handoff

---

## 📊 上下文阈值

| 使用率 | 动作 | 说明 |
|--------|------|------|
| **<60%** | 正常 | 继续对话 |
| **60-70%** | 提醒 | 告知用户当前状态 |
| **70-80%** | 建议 | 建议 handoff 到新会话 |
| **>80%** | 强制 | 生成 handoff 文档，结束当前会话 |

---

## 🔄 Handoff 流程

### 触发条件
- [ ] 任务阶段完成（Phase 1/2/3）
- [ ] 上下文 > 70%
- [ ] 会话时长 > 2 小时
- [ ] 用户主动请求

### Handoff 文档模板

```markdown
# {任务名称} - 会话交接

**交接时间**: YYYY-MM-DD HH:mm
**上一会话**: {sessionKey}

## ✅ 已完成
- 任务 1
- 任务 2

## 🟡 进行中
- 当前步骤
- 待验证

## ⏸️ 下一步
1. 行动 1
2. 行动 2

## 📁 关键文件
- `/path/to/file1`
- `/path/to/file2`

## 🔧 配置状态
- API Key: ✅ 已配置
- 技能部署：✅ 已完成
```

### 继续指南

```bash
# 在新会话中
1. 读取 HANDOVER.md
2. 确认任务状态
3. 继续下一步
```

---

## 🧹 自动清理策略

### 图片管理
- **限制**: 最多 5 张
- **策略**: 保留最近 3 张，旧图片压缩为文字描述
- **触发**: 第 6 张图片上传时

### 代码管理
- **限制**: 最多 10 段
- **策略**: 折叠为文件引用 `see: /path/to/file.js`
- **触发**: 第 11 段代码时

### 历史管理
- **限制**: 最多 100 条消息
- **策略**: 生成阶段摘要，保留最近 50 条
- **触发**: 第 101 条消息时

---

## 📝 用户操作指南

### 日常使用
```
正常对话 → 自动管理上下文
```

### 长周期任务
```
开始任务 → 创建任务文档 → 分阶段进行 → 每阶段 handoff
```

### 超限处理
```
收到提醒 → 选择继续/ handoff → 生成交接文档 → 新会话继续
```

---

## 🎯 最佳实践

### ✅ 推荐
- 每个任务创建独立文档
- 每 2 小时或每阶段 handoff
- 关键决策写入 MEMORY.md
- 使用 `/status` 检查会话状态

### ❌ 避免
- 单会话超过 6 小时
- 不记录关键决策
- 忽略上下文提醒
- 在群聊中分享敏感信息
