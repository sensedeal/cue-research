## 🗺️ 用户体验旅程

### 阶段 1: 安装与配置 (5 分钟)

#### 步骤 1.1: 安装 Skill

**用户操作:**
```bash
openclaw skills install https://github.com/sensedeal/cue-research.git
```

**预期体验:**
- ✅ 下载成功
- ✅ 安装依赖
- ✅ 提示配置 API Key

**测试检查点:**
```bash
# 1. 检查 skill 是否安装
ls ~/.openclaw/workspace/skills/cue-research/

# 2. 检查依赖是否安装
ls ~/.openclaw/workspace/skills/cue-research/node_modules/

# 3. 检查是否提示配置
# 应该看到："请配置 CUECUE_API_KEY"
```

**可能的问题:**
- ❌ 依赖安装失败
- ❌ 没有配置提示
- ❌ 权限问题

---

**预期体验:**

**用户操作:**


**预期体验:**

- ✅ 列出所有可用命令
- ✅ 有使用示例
- ✅ 有快捷用法说明

```bash
# □ 包含 /ct 命令说明
# □ 包含 /cm 命令说明
```

- ❌ 缺少示例

---

#### 步骤 2.2: 发起第一个调研

**用户操作:**
```
/cue 分析宁德时代竞争优势
```

**预期体验:**
- ✅ 立即响应 (3 秒内)
- ✅ 显示任务已启动
- ✅ 显示研究视角 (如"产业研究视角")
- ✅ 显示报告链接
- ✅ 显示预计耗时
- ✅ 承诺完成后通知

**测试检查点:**
```bash
# 1. 发送调研命令
/cue 分析宁德时代竞争优势

# 2. 检查响应时间
# 应该 < 3 秒

# 3. 检查响应内容：
# □ "🚀 研究任务已启动"
# □ 主题：分析宁德时代竞争优势
# □ 视角：产业研究视角 (或其他检测到的模式)
# □ 链接：https://cuecue.cn/c/xxx
# □ 预计耗时：5-30 分钟
# □ "完成后会自动通知你"

# 4. 检查任务文件是否创建
ls ~/.openclaw/workspaces/feishu-{user_id}/.cuecue/tasks/
# 应该有一个新的 task_*.json 文件

# 5. 检查任务文件内容：
cat task_*.json
# 应该包含：taskId, topic, status: running, progress, conversationId, reportUrl
```

**可能的问题:**
- ❌ 响应超时 (>10 秒)
- ❌ 没有显示视角
- ❌ 没有报告链接
- ❌ 任务文件未创建
- ❌ API Key 错误提示不友好

### 阶段 3: 等待与查询 (5-30 分钟)

#### 步骤 3.1: 查询任务状态

**用户操作:**
```
/ct
```

**预期体验:**
- ✅ 显示当前任务列表
- ✅ 每个任务显示：
  - 主题
  - 当前进度
  - 开始时间
  - 预计剩余时间
  - 取消按钮
- ✅ 如果没有任务，友好提示

**测试检查点:**
```bash
# 1. 发送状态查询
/ct

# 2. 检查响应内容：
# □ 显示任务主题
# □ 显示当前进度 (如"正在搜索信息...")
# □ 显示开始时间
# □ 显示进度百分比 (可选)
# □ 有"取消任务"按钮

# 3. 如果没有任务：
# □ 显示"暂无进行中的任务"
# □ 提示如何发起新任务
```

**可能的问题:**
- ❌ 返回占位文本 (当前代码问题)
- ❌ 不显示进度
- ❌ 没有取消按钮
- ❌ 显示错误

## 📊 测试评分标准

### 体验评分 (1-5 分)

| 维度 | 5 分 | 3 分 | 1 分 |
|------|------|------|------|
| **响应速度** | <3 秒 | 3-10 秒 | >10 秒 |
| **错误提示** | 清晰友好 | 能看懂 | 看不懂/无提示 |
| **功能完整** | 全部可用 | 部分可用 | 大部分不可用 |
| **交互流畅** | 按钮都响应 | 部分响应 | 无响应 |
| **通知及时** | 主动推送 | 需手动查询 | 无通知 |
