# 🧪 完整客户旅程测试方案

**测试视角:** 真实用户 + 真实对话场景  
**设计原则:** 不是测试功能，而是测试用户体验

---

## 👥 用户画像与场景

### 用户类型 1: 新手用户 (第一次使用)

**特征:**
- 不知道有什么功能
- 不会用命令
- 会问各种问题
- 容易困惑

**典型对话流程:**
```
用户："这是什么？"
Bot:  [介绍功能]

用户："怎么用？"
Bot:  [使用指南]

用户："帮我看看宁德时代"
Bot:  [启动调研]

用户："好了吗？"
Bot:  [显示进度]

用户："太慢了"
Bot:  [安抚 + 预计时间]

[5 分钟后]
Bot:  [推送完成通知]

用户："不错，帮我盯着这个"
Bot:  [创建监控]
```

**测试检查点:**
- [ ] 新手引导是否清晰
- [ ] 自然语言是否理解
- [ ] 等待时是否有进度反馈
- [ ] 完成通知是否主动
- [ ] 监控创建是否简单

---

### 用户类型 2: 专业用户 (每天使用)

**特征:**
- 知道有什么功能
- 会用简洁表达
- 追求效率
- 会同时发起多个任务

**典型对话流程:**
```
用户："/cue 宁德时代财报分析”
Bot:  [启动]

用户："/cue 比亚迪竞争力”
Bot:  [启动]

用户："/ct"
Bot:  [显示 2 个任务]

用户："第一个好了吗？"
Bot:  [显示进度 80%]

[完成后]
用户："开启监控”
Bot:  [创建]

用户："/cm"
Bot:  [显示监控列表]
```

**测试检查点:**
- [ ] 多任务并发支持
- [ ] 任务状态查询准确
- [ ] 监控管理方便
- [ ] 命令响应快速

---

### 用户类型 3: 犹豫用户 (不确定要不要用)

**特征:**
- 会试探性提问
- 容易放弃
- 需要鼓励
- 可能会取消

**典型对话流程:**
```
用户："你能做什么？"
Bot:  [介绍]

用户："分析一下 AI 行业"
Bot:  [启动]

[等待中...]
用户："算了，不要了"
Bot:  [友好取消]

[几小时后]
用户："刚才那个分析好了吗？"
Bot:  [显示已完成]
```

**测试检查点:**
- [ ] 取消是否友好
- [ ] 完成后是否还能查看
- [ ] 是否有挽回机制

---

## 🗺️ 完整客户旅程地图

### 阶段 1: 发现与尝试 (0-5 分钟)

#### 场景 1.1: 第一次看到

**用户心理:** "这是什么东西？靠谱吗？"

**可能的对话:**
```
"这是什么？"
"你能干嘛？"
"怎么用？"
"收费吗？"
```

**测试重点:**
- [ ] 帮助菜单是否清晰
- [ ] 是否有使用示例
- [ ] 是否降低使用门槛

---

#### 场景 1.2: 第一次尝试

**用户心理:** "试试看，不好用就放弃"

**可能的对话:**
```
"帮我分析宁德时代"
"研究一下 AI"
"看看腾讯怎么样"
```

**测试重点:**
- [ ] NLU 是否准确识别
- [ ] 响应是否快速 (<3 秒)
- [ ] 是否显示预计时间
- [ ] 是否承诺完成通知

**失败场景:**
```
用户："分析宁德"
Bot:  [无响应或报错]
→ 用户流失
```

---

### 阶段 2: 等待与焦虑 (5-30 分钟)

#### 场景 2.1: 主动查询

**用户心理:** "怎么还没好？卡住了吗？"

**可能的对话:**
```
"好了吗？"
"进度怎么样？"
"还要多久？"
"怎么这么慢"
```

**测试重点:**
- [ ] `/ct` 是否显示真实进度
- [ ] 是否有预计剩余时间
- [ ] 是否有安抚文案
- [ ] 是否提供取消选项

**失败场景:**
```
用户："好了吗？"
Bot:  "📊 任务状态查询功能正常，具体数据接入存储层即可。"
→ 用户困惑：所以是好了还是没好？
```

---

#### 场景 2.2: 失去耐心

**用户心理:** "太慢了，不要了"

**可能的对话:**
```
"取消"
"不要了"
"停下"
```

**测试重点:**
- [ ] 是否识别取消意图
- [ ] 取消是否成功
- [ ] 取消后是否友好

---

### 阶段 3: 接收结果 (5-30 分钟后)

#### 场景 3.1: 主动推送

**用户心理:** "哦？好了？看看怎么样"

**预期通知:**
```
✅ 研究完成：宁德时代竞争优势

摘要：
宁德时代在全球动力电池市场份额连续 6 年第一...

核心观点：
1. 技术优势：CTP 技术领先行业 2-3 年
2. 成本优势：规模效应 + 垂直整合
3. 客户优势：绑定特斯拉、宝马等头部车企

🔗 查看详细报告：[链接]

[开启持续监控] [分享]
```

**测试重点:**
- [ ] 是否主动推送
- [ ] 是否有摘要 (不是只有链接)
- [ ] 是否有核心观点
- [ ] 是否有操作按钮

**失败场景:**
```
Bot:  [无通知]
用户：[不知道完成，体验极差]
```

---

#### 场景 3.2: 查看报告

**用户心理:** "写得怎么样？有用吗？"

**可能的对话:**
```
"不错，帮我盯着这个"
"一般般"
"太简单了"
```

**测试重点:**
- [ ] 监控创建是否简单
- [ ] 是否有延伸推荐
- [ ] 是否支持追问

---

### 阶段 4: 长期使用 (数天 - 数周)

#### 场景 4.1: 监控通知

**用户心理:** "有什么新动态？看看"

**预期通知:**
```
🔔 监控触发：固态电池

丰田宣布固态电池量产时间提前至 2027 年

摘要：
丰田汽车今日宣布，其固态电池研发取得重大突破...

🔗 查看详情：[链接]
⏰ 触发时间：2026-03-12 10:30

[查看详情] [暂停监控]
```

**测试重点:**
- [ ] 是否主动推送
- [ ] 内容是否有价值
- [ ] 是否有原文链接
- [ ] 是否有管理选项

---

#### 场景 4.2: 管理监控

**用户心理:** "监控太多了，整理一下"

**可能的对话:**
```
"我有哪些监控？"
"关掉固态电池的监控"
"删除那个"
```

**测试重点:**
- [ ] 列表是否清晰
- [ ] 启停是否方便
- [ ] 删除是否确认

## 🎯 测试执行方法

### 方法 1: 角色扮演测试

**执行步骤:**
1. 测试人员扮演真实用户
2. 按照用户画像对话
3. 记录每一步体验
4. 标记困惑点

**记录模板:**
```
场景：新手用户第一次尝试
时间：2026-03-12 10:00
用户输入："帮我分析宁德时代"
Bot 响应：[记录实际响应]
响应时间：2 秒
用户体验：✅ 清晰 / ❌ 困惑
问题记录：[如有]
```

---

### 方法 2: 真实用户测试

**执行步骤:**
1. 邀请 3-5 位真实用户
2. 不给使用说明
3. 观察自然使用过程
4. 记录卡点和疑问

**观察要点:**
- 第一次输入什么
- 哪里犹豫了
- 哪里困惑了
- 是否完成核心任务

---

### 方法 3: 对话日志分析

**执行步骤:**
1. 收集真实对话日志
2. 分析常见问法
3. 识别失败场景
4. 优化响应

**分析维度:**
- 用户常用表达
- 识别失败案例
- 取消率高的场景
- 用户流失点

## 📊 成功指标

### 核心指标

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| **首次响应时间** | <3 秒 | 日志统计 |
| **NLU 识别准确率** | >90% | 抽样检查 |
| **任务完成率** | >80% | 完成数/发起数 |
| **用户留存率** | >50% | 7 天后仍使用 |
| **监控创建率** | >30% | 创建监控/完成任务 |
