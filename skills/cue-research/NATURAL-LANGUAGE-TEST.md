## 🗺️ 自然语言交互旅程

### 阶段 1: 初次相遇 (5 分钟)

#### 场景 1.1: 用户第一次尝试

**用户说:**
```
帮我分析宁德时代竞争优势
```

**预期体验:**
- ✅ 立即识别调研意图
- ✅ 显示任务已启动
- ✅ 显示研究视角
- ✅ 显示报告链接
- ✅ 预计耗时
- ✅ 承诺完成后通知

**测试检查点:**
```bash
# 1. 发送自然语言
帮我分析宁德时代竞争优势

# 2. 检查响应时间 < 3 秒

# 3. 检查响应内容：
# □ "🚀 研究任务已启动"
# □ 主题：分析宁德时代竞争优势
# □ 视角：产业研究视角
# □ 链接：https://cuecue.cn/c/xxx
# □ 预计耗时：5-30 分钟

# 4. 检查任务文件创建
```

**如果失败:**
- ❌ 没有识别意图 → 传递给底层大模型
- ❌ 响应超时 → 用户失去耐心
- ❌ 没有报告链接 → 用户不知道在哪看

---

#### 场景 1.2: 用户不确定是否成功

**用户说:**
```
好了吗？
```
或
```
进度怎么样？
```

**预期体验:**
- ✅ 识别为查询进度
- ✅ 显示当前任务状态
- ✅ 显示进度百分比
- ✅ 显示预计剩余时间

**测试检查点:**
```bash
# 1. 发送查询
好了吗？

# 2. 检查响应：
# □ 显示任务主题
# □ 显示当前进度 (如"正在搜索信息...")
# □ 显示进度百分比 (如"30%")
# □ 显示预计剩余时间 (如"还需 10 分钟")
```

**如果失败:**
- ❌ 没有识别 → 用户困惑
- ❌ 返回占位文本 → 用户失望
- ❌ 没有进度 → 用户不知道还要等多久

---

### 阶段 2: 等待过程 (5-30 分钟)

#### 场景 2.1: 用户等不及了

**用户说:**
```
取消吧
```
或
```
不要了
```

**预期体验:**
- ✅ 识别为取消任务
- ✅ 显示要取消的任务
- ✅ 确认后取消
- ✅ 返回取消成功

**测试检查点:**
```bash
# 1. 发送取消
取消吧

# 2. 检查响应：
# □ 显示可取消的任务
# □ 有确认按钮

# 3. 确认后：
# □ "✅ 任务已取消"
# □ 任务状态更新
```

**如果失败:**
- ❌ 没有识别 → 用户困惑
- ❌ 命令未实现 → 用户沮丧
- ❌ 取消后还能看到 → 数据不一致

### 阶段 3: 接收结果 (5-30 分钟后)

#### 场景 3.1: 主动推送完成通知

**预期体验:**
- ✅ 主动推送 (无需用户问)
- ✅ 包含摘要
- ✅ 包含核心观点
- ✅ 包含报告链接
- ✅ 有"开启监控"按钮

**用户看到:**
```
✅ 研究完成：宁德时代竞争优势

摘要：
宁德时代在全球动力电池市场份额连续 6 年第一...

核心观点：
1. 技术优势：CTP 技术领先行业 2-3 年
2. 成本优势：规模效应 + 垂直整合
3. 客户优势：绑定特斯拉、宝马等头部车企

🔗 查看详细报告：https://cuecue.cn/c/xxx

[开启持续监控] [分享]
```

**测试检查点:**
```bash
# 1. 等待完成通知
# 应该在 5-30 分钟内主动推送

# 2. 检查通知内容：
# □ 有摘要 (不是只有链接)
# □ 有核心观点列表
# □ 有完整报告链接
# □ 有操作按钮

# 3. 点击"开启监控":
# □ 创建监控
# □ 返回确认
```

**如果失败:**
- ❌ 没有主动推送 → 用户不知道完成
- ❌ 只有链接 → 用户体验差
- ❌ 按钮无响应 → 用户困惑
