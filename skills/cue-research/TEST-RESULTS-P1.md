## ℹ️ 不需要实现的功能

### `cancel_task` 按钮

**原因:**
根据旧版代码和架构设计：

1. **任务在服务端运行** - CueCue API 执行调研，本地无法取消
2. **旧版取消功能** - 只是清理本地卡住的任务（>30 分钟无更新）
3. **新版架构** - 任务状态由服务端管理，本地只负责展示

**旧版实现参考:**
```javascript
// 旧版 handleCancel() 只是清理本地卡住的任务
async function handleCancel() {
  const cleanedCount = await cleanupStuckTasks(); // 清理>30 分钟无更新的任务
  return `✅ 已清理 ${cleanedCount} 个卡住的任务`;
}
```

**新版决策:**
- ❌ 不实现 `cancel_task` 按钮
- ✅ 任务完成由服务端主动推送
- ✅ 本地只负责展示状态
